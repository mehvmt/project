PROJECT: BD Grup Makina — Corporate Website  
(Language: Turkish content, but instructions in English)

Summary:  
Create a professional and modern corporate website for **BD Grup Makina**.  
The theme should primarily use **dark blue and gray** tones.  
The site must be **responsive, fast, and accessible**, built only with **HTML5 and CSS3** (minimal JavaScript allowed only for lightbox or simple transitions).  
Include the company logo (`logo.svg`) in both the header and footer.  
A **gallery section (catalog)** should display **at least 20 images** comfortably.  
The code must be clean, easy to read, and simple to edit later.

---

### General Requirements:
- Use **only HTML5 + CSS3**, no frameworks (JS only if absolutely necessary).  
- Fully **responsive** (mobile-first layout).  
- **Semantic HTML** structure (header, main, section, footer).  
- Use **CSS variables** (`:root`) for colors and typography to allow easy customization.  
- Optimized images stored in `/assets/gallery/` (gallery-01.jpg ... gallery-20.jpg).  
- All code must be properly **commented and well-organized**.  
- The final version should be easily deployable as a static website.

---

### Page Structure:

#### 1. Home (Hero Section)
- Large hero area: text on the left, image or slider on the right.  
- Main title: “BD Grup Makina — Industrial and Custom Engineering Solutions”  
- Short subtitle (2 lines max).  
- Two call-to-action buttons: “View Projects” (scroll to gallery) and “Contact Us” (scroll to contact).  
- Smooth fade-in or slide-in CSS-only animations.  
- Optionally include a light parallax background or gradient overlay.

#### 2. About / Company Info
- 2–3 paragraphs of placeholder text about the company.  
- Optional area for certifications or partner logos.

#### 3. Services / Products
- 3–6 service cards (each with an icon, title, and short description).  
- Hover effects: smooth shadow or slight scale-up.  
- Use a responsive grid layout.

#### 4. Gallery / Project Catalog
- Responsive grid layout:
  - Desktop: 4 columns  
  - Tablet: 3 columns  
  - Mobile: 1–2 columns  
- Display **at least 20 images**.  
- Each image should show a **hover overlay** with project title.  
- Clicking opens a **simple CSS (or minimal JS) lightbox**.  
- Images located in `/assets/gallery/`.  
- Clean spacing and modern layout with balanced padding.

#### 5. Contact
- Display email and phone clearly (`mailto:` and `tel:` links).  
- Simple contact form:
  - Name, Email, Phone, Message fields.  
  - Submit button with hover animation.  
- Minimal form validation (HTML5 built-in).  
- Include a Google Maps iframe placeholder below the form.

#### 6. Footer
- Left: logo (`logo.svg`)  
- Center: address or short info  
- Right: email, phone, and social icons  
- Background: dark blue  
- Text: white or light gray  
- Include a copyright line.

---

### Design & Visual Guidelines:
**Color Palette**
- Primary (dark blue): `#0A3D91`  
- Secondary (gray): `#6B7280`  
- Background light gray: `#F3F4F6`  
- White: `#FFFFFF`  
- Accent (button hover): `#08306F`  

**Typography**
- Headings: Poppins or Montserrat  
- Body text: Inter or Roboto  
- Responsive font sizing (H1 ~36–48px desktop, body ~16px)

**Animations & Transitions**
- Use **CSS transitions or keyframes only** (no JS libraries).  
- Page elements: fade-in or slide-in on scroll.  
- Buttons: scale(1.03) + shadow on hover.  
- Images: slight zoom (1.05x) on hover.  
- Transition duration: 0.3–0.5s, `ease-out`.

---

### Accessibility & SEO
- All images must include descriptive `alt` text.  
- Proper heading hierarchy (h1–h6).  
- Include `<meta name="viewport">` for mobile scaling.  
- Include meta title, description, and Open Graph tags (use placeholders).  

---

### Performance
- Use **WebP** image format where possible.  
- One main CSS file (`style.css`), minifiable.  
- Avoid unnecessary JavaScript.  
- Optimized and lazy-loaded images.
